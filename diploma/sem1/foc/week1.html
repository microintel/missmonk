The four systems

One idea, four alphabets

Every number system works the same way: a digit's value depends on which digit it is and where it sits. The only thing that changes is how many symbols are available before you have to carry over — that count is called the base. Tap the digits below to see each system's own alphabet.

Binary BASE 2

  • Only two digits exist: 0 and 1
  • Matches a switch: off / on
  • Every digital computer stores data this way
  • One digit = one bit; 4 bits = a nibble; 8 bits = a byte
Example number: 1101₂

Octal BASE 8

  • Eight digits: 0 to 7
  • Each digit needs exactly 3 bits
  • Used as shorthand for long binary strings
  • Rarely typed by hand today, but common in older systems
Example number: 257₈

Decimal BASE 10

  • Ten digits: 0 to 9 — the one you already use
  • Also called the Hindu-Arabic system
  • Grew from counting on ten fingers
  • Easiest for humans to read; hardest for circuits to store
Example number: 205₁₀

Hexadecimal BASE 16

  • Sixteen digits: 0–9, then A–F for 10–15
  • Each digit needs exactly 4 bits (one nibble)
  • Written with a 0x prefix or h suffix
  • Used to write long binary numbers compactly — colours, memory addresses
Example number: 1AF₁₆

Turning digits into letters

Computer codes

Number systems represent quantities. But your keyboard needs to send letters, punctuation and symbols too — so computers assign each character its own binary pattern. Here are the codes you'll meet most often.

BCD — Binary Coded Decimal (the 8421 code)

Each decimal digit (0–9) gets its own 4-bit binary pattern. Four bits can technically make 16 patterns, but BCD only uses the first ten — 1010 through 1111 are invalid.

ASCII — American Standard Code for Information Interchange

ASCII gives every keyboard character a number. Basic ASCII (7 bits) covers 128 characters; Extended ASCII (8 bits) covers 256. Type a single character to see its code.

Only works for standard keyboard characters (basic ASCII range 0–127).

Unicode — one code for every language

ASCII only covers English. Unicode assigns a code point to characters from every written language, plus symbols and emoji, and can be stored as UTF-8, UTF-16 or UTF-32. Try a character — including non-English ones.

What to remember

Core vocabulary

  • Bit — one binary digit (0 or 1)
  • Nibble — 4 bits
  • Byte — 8 bits
  • MSB / LSB — Most / Least Significant Bit (leftmost / rightmost)
  • Base — how many digits a system has before carrying

Converting to another base

  • Integer part: divide by the new base repeatedly, read remainders bottom-to-top
  • Fraction part: multiply by the new base repeatedly, read carries top-to-bottom
  • Stop dividing once the quotient is 0; stop multiplying once you reach the accuracy you need

Converting from another base to decimal

  • Multiply each digit by (base ^ position)
  • Positions count from 0 at the point, increasing left, decreasing (negative) right
  • Add every product together

Codes in one line each

  • BCD — each decimal digit → its own 4 bits
  • ASCII — each keyboard character → 7 or 8 bits
  • EBCDIC — IBM's 8-bit character code
  • Unicode — one standard for every language on earth

Frequently asked

Why do computers use binary instead of decimal?

Because a transistor is easiest to build with two reliable states — current flowing or not. Two states map perfectly onto 0 and 1, which is why binary became the natural language of digital circuits.

What's the difference between octal/hex and binary?

They're not really different data — they're different-sized "chunks" of the same binary. Octal groups bits in 3s, hex groups bits in 4s, purely so long binary strings are shorter and easier for humans to read and type.

Why does BCD "waste" six combinations?

Four bits can make 16 patterns (0000–1111), but decimal only has ten digits (0–9). The six leftover patterns (1010–1111) simply aren't assigned a meaning in BCD, so they're considered invalid.

Is Unicode replacing ASCII?

Not exactly — Unicode's UTF-8 encoding was designed so that plain English text is stored identically to ASCII. So ASCII text is already valid UTF-8; Unicode simply extends the range to cover every other language too.
